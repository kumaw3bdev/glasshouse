Case Study: Digital Infrastructure & Accessibility Remediation

Project: Glass House Bar Menu System

Lead Operations & Accessibility: Damascus Mosley (Mosley Automation Systems) 

Executive Summary

This project involved the total remediation of a legacy digital menu system into a high-performance, WCAG 2.1 AA compliant infrastructure. By treating accessibility and performance as core operational requirements, I mitigated legal risk and improved system availability for all patrons.


Technical Implementation

Semantic HTML5 Architecture: 
Eliminated "div-soup" in favor of standardized landmarks (nav, main, section, etc).

Infrastructure: 
Provisioned a version-controlled deployment pipeline via GitHub Pages for 100% uptime.

Asset Optimization: 
Implemented a strict 49kb budget for core imagery to ensure usability in high-pressure bar environments with degraded mobile signals.

Zero-Dependency Design: 
Removed all external script dependencies to ensure the "Source of Truth" is entirely self-contained and auditable.

Accessibility & Compliance (a11y)

I view accessibility as a systems issue. If a patron cannot access the menu due to a screen reader failure, it is an operational failure.

WAVE Audit Result: 0 Errors / 0 Contrast Errors.

Manual Validation: Verified keyboard navigation focus paths and ARIA label integrity.

Risk Mitigation: Provided the client with technical proof of conformance to reduce ADA litigation exposure.

Operational Controls (The Mosley Standard)
To prevent configuration drift, the following controls were implemented:

Change Log: An immutable audit trail for all system modifications.

Post-Deployment Checklist: Standardized Operating Procedure (SOP) for content updates.

Monthly Compliance Audit: Scheduled recurring system health checks to verify performance and security.
